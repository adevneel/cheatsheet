Form Cheatsheet

Elements

Element Name Description
<form> Form Used to collect and send information to an external source.
Requires action and method attributes.
<input> Input Used to render a variety of input fields on a webpage including text fields, checkboxes, buttons, etc.
Requires type id and name attributes.
The for attribute is also used for the <datalist> element.
<textarea> Textarea Used when creating a text-box for multi-line input (e.g. a comment section).
Requires the name attribute and supports the rows and cols attributes which determine the height and width of the element.
<select> Select Used to create a dropdown list. A list of choices for the dropdown list can be created using one or more option elements.
Requires the name attribute and the option element requires a value attribute.
<datalist> Datalist Used to achieve a basic search/autocomplete functionality, with the list of data being shown as a dropdown. As the user starts typing, the list will be updated to show elements that best match what has been typed into the input field. The actual list items are specified as multiple option elements nested inside the datalist.
Requires an id attribute that matches the value of the <input>'s list attribute, with the option elements requiring value attributes.
<label> Label Provides identification for a specific <input> based on matching values of the <input>’s id attribute and the <label>’s for attribute.
Requires a for attribute that matches the associated element's id attribute.

Attributes

Attribute Association Description
name <input>, <textarea>, <select> In order for a form to send data, it needs to be able to put it into key-value pairs. This is achieved by setting the name attribute of the input element.
required <input> Specifies that the field must include a value; can't be left blank.
max <input type="number"> Specifies the maximum value for the input fieldin number type <input> elements.
min <input type="number"> Specifies the minimum value for the input field in number type <input> elements.
maxlength <input type="text"> Specifies the maximum number of characters that can be entered into the field in text type <input> elements.
minlength <input type="text"> Specifies the minimum number of characters that can be entered into the field in text type <input> elements.
pattern <input type="text"> Uses a regular expression to match against (or validate) the value of the <input>, when the form is submitted.

Input Types

Input Type Name Description
type="text" Text A single row input field that users can type text inside.
type="checkbox" Checkbox A single checkbox item; users can select single or multiple options.
Requires including a value attribute.
type="number" Number An input field that allows the user to enter only numbers and a few special characters inside the field.
Can include a step attribute which creates arrows inside the input field to increase or decrease by the value of the step attribute.
type="range" Range A range slider that acts as a selector between a min and a max value.
Can include a step attribute which createsarrows inside the input field to increase or decrease by the value of the step attribute.
type="radio" Radio This renders a single radio button; only a single option can be selected from a group of radio buttons.
Requires including a value attribute.
type="password" Password A single row input field which allows the user to type censored text inside the field.
Can include a pattern attribute, which is used to match against (or validate) the value of the <input> when the form is submitted.

Submit Button

HTML <input> elements can have a type attribute set to submit, by adding type="submit". With this attribute included, a submit button will be rendered and, by default, will submit the <form> and execute its action.

The text of a submit button is set to submit by default but can also be changed by modifying the value attribute.
